Team improvement

Helped a rapidly growing software development team introduce processes to improve quality and employee onboarding.

Highlights

  • Code reviews
    Introduced code reviews before code was merged into main branch to help early identification of issues, and to increase system knowledge between developers.
  • Development virtual machines
    Created a development virtual machine set up to increase onboarding speed for new employees. It also made the development environments consistent for all developers, reducing issues caused by differences between production and development environments.
  • Deployment process
    Moved from a completely manual process of updating source code on the production servers for each customer to a single codebase and scripted deployment. Later extended to a deployment pipeline using Jenkins which allowed non-developers to make deployments and offered visibility for automated test results.
  • Subversion to Git migration
    Migrated all company projects from an internally managed Subversion server to GitHub. Moving to Git made merging branches for larger numbers of developers easier.
